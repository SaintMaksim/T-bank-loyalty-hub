# Техническое задание (ТЗ) на проект «Единый раздел лояльности Т-Банка»

## 1. Введение

### 1.1. Наименование системы

Единый раздел лояльности Т-Банка (Loyalty Hub)

### 1.2. Основание для разработки

Хакатон по кейсу Т-Банка: необходимо объединить разрозненные инструменты лояльности (кэшбэк, акции, подписки) в один раздел, показывающий пользователю совокупную выгоду и стимулирующий активность.

### 1.3. Цели и задачи

- Предоставить пользователю прозрачную информацию о накопленном кэшбэке в рублях, милях и баллах «Браво».
- Дать прогноз будущей выгоды на основе истории начислений.
- Персонализировать акции партнёров и кросс-продукты экосистемы под финансовый сегмент (LOW/MEDIUM/HIGH).
- Удерживать клиента с помощью геймификационных механик: уровни, стрики, ачивки, челленджи.
- Внедрить полноценного ИИ-советника на базе LLM для генерации живых, персонализированных финансовых рекомендаций.

## 3. Функциональные требования

### 3.1. Модуль аналитики

Все аналитические выкладки оформлены в папке `/docs` и являются основой для разработки:

| Файл                 | Содержание                                                                                                                     |
| :------------------- | :----------------------------------------------------------------------------------------------------------------------------- |
| `data-mapping.md`    | Структура и связи БД, аномалии.                                                                                                |
| `user-segments.md`   | Сегментация по суммарному балансу счетов: LOW (<500k), MEDIUM (500k–5M), HIGH (>5M).                                           |
| `loyalty-metrics.md` | Формулы: `total_equivalent_rub = rub + bravo*0.5 + miles*2`. Прогноз: среднее за 3 мес \* 1.2.                                 |
| `hypotheses.md`      | 6 UI-гипотез (главная метрика вверху, эмоциональный прогноз, геймификация, генеративный ИИ-совет).                             |
| `api-contract.md`    | JSON-контракты всех эндпоинтов.                                                                                                |
| `gamification.md`    | Описание механик геймификации.                                                                                                 |
| `ai-feature.md`      | Описание интеграции с Claude 3.5 Haiku: структура системного промпта, формирование контекста (балансы, прогноз, топ-3 оффера). |

### 3.2. Бэкенд

#### 3.2.1. Эндпоинты

| Метод  | Эндпоинт                     | Описание                                                                                           |
| :----- | :--------------------------- | :------------------------------------------------------------------------------------------------- |
| `GET`  | `/users/`                    | Список всех пользователей                                                                          |
| `GET`  | `/users/me`                  | Данные текущего авторизованного пользователя                                                       |
| `GET`  | `/loyalty/summary`           | Совокупная выгода (баланс в рублях, милях, баллах)                                                 |
| `GET`  | `/loyalty/forecast`          | Прогноз будущей выгоды                                                                             |
| `GET`  | `/loyalty/history`           | Полная история начислений лояльности                                                               |
| `GET`  | `/loyalty/history/monthly`   | Агрегированная помесячная история начислений                                                       |
| `GET`  | `/offers/`                   | Персональные акции партнёров (с фильтром по сегменту)                                              |
| `GET`  | `/ai/recommend`              | Генерация персонального ИИ-совета через Anthropic API на основе агрегированных данных пользователя |
| `GET`  | `/gamification/streak`       | Текущий стрик пользователя и награды                                                               |
| `POST` | `/gamification/streak/visit` | Отметить ежедневное посещение (обновление стрика)                                                  |

_(Примечание: пути эндпоинтов унифицированы для работы с сессией/токеном текущего пользователя `current_user`)._

#### 3.2.2. Логика расчётов

- `total_equivalent_rub` – по формуле из `loyalty-metrics.md`.
- **Прогноз** – среднее за последние 3 месяца × 1.2.
- **Фильтрация акций** (HIGH → все; MEDIUM → MEDIUM + LOW; LOW → только LOW), сортировка по `cashback_percent`.
- **Генерация ИИ-советов**: Бэкенд собирает контекст (накопленные рубли, мили, баллы, прогноз и топ-3 доступных акции) и отправляет запрос к модели `claude-3-5-haiku` с жестким системным промптом для получения короткого (2-3 предложения) мотивирующего совета.

#### 3.2.3. Тестирование (минимум 5 тест-кейсов)

Примеры (pytest):

1.  Корректность `total_equivalent_rub` для пользователя с разными валютами.
2.  Фильтрация офферов для сегмента MEDIUM.
3.  Прогноз неотрицательный и умножается на 1.2.
4.  Успешный маппинг Pydantic-схем при отдаче истории.
5.  Обработка ошибки (fallback текст), если API Claude недоступно.

### 3.3. Фронтенд

#### 3.3.1. Страницы и виджеты

Одна страница раздела лояльности, состоит из виджетов (сверху вниз):

1. **Шапка** – выпадающий список для выбора тестового пользователя, иконка переключения темы.
2. **Карточка «Ваша выгода»** – крупно `total_equivalent_rub` + breakdown (рубли, мили, баллы).
3. **Прогноз** – текстовая строка с суммой.
4. **Геймификация**:
   - Прогресс-бар уровня с названием (Бронза/Серебро/Золото/Платина).
   - Календарь стриков (иконка и текущая длина).
   - Текущий бустер (если активен).
   - Задание дня с кнопкой «Выполнить».
   - Сетка из 3–4 последних ачивок.
5. **Акции партнёров** – горизонтальный скролл карточек (логотип-заглушка, название, % кэшбэка, цвет бренда).
6. **Кросс-селл** – 2–3 плитки с продуктами Т‑Инвестиции, Т‑Бизнес, Т‑Мобайл (фильтр по сегменту).
7. **ИИ-совет** – блок с текстом и кнопкой «Обновить».
8. **Таблица лидеров** – вкладки «Мой сегмент» и «Все», отображается место пользователя и топ-5.
9. **Колесо фортуны** – кнопка, анимация, модальное окно с призом.

#### 3.3.2. Технические требования

- Адаптивная вёрстка (мобильные до 480px, планшеты, десктоп).
- Поддержка тёмной темы (переключение через кнопку или системное).
- Работа с API через `fetch` / `axios`.
- Выполненные челленджи, стрики и вращения колеса сохраняются в `localStorage` или синхронизируются с бэкендом (на усмотрение команды).

#### 3.3.3. Дополнительно

- График истории начислений (опционально, на основе `/history`) – линейный график с Recharts.

### 3.4. Мобильное приложение

- Упаковка веб-приложения в Android APK.
- Настройка иконок, сплеш-экрана.
- Собранный `.apk` файл размещается в корне репозитория или по ссылке (указывается в README).

### 3.5. CI/CD и деплой

GitHub Actions настроен на:

- Запуск тестов бэкенда (pytest) при pull request в `main`.
- Проверку сборки фронтенда (`npm run build`).

**Деплой:**

- Бэкенд и База данных – развернуты в Docker-контейнерах на выделенном европейском VPS/VDS сервере (для беспрепятственного доступа к зарубежным API).
- Фронтенд – на Vercel или Netlify.
- Ссылки на работающие приложения указываются в `README`.

### 3.6. Требования к дизайну

#### 3.6.1. Общий стиль

- Визуально соответствовать экосистеме Т-Банка: светлые фоны, скруглённые углы (8–12px), зелёный цвет акцентов (например, #00B4D8 или согласовать с командой).
- Карточки виджетов – белые (светлая тема) / тёмно-серые (тёмная тема) с лёгкой тенью.
- Иконки – линейные или заливные, единая толщина (2px).

#### 3.6.2. Обязательные элементы дизайна

| Элемент                | Требования                                                                                             |
| ---------------------- | ------------------------------------------------------------------------------------------------------ |
| Шапка                  | Верхняя полоса: выбор пользователя (селект), иконка темы (луна/солнце).                                |
| Карточка «Ваша выгода» | Крупный шрифт для `total_equivalent_rub` (32–40px). Три строки с иконками валют и суммами.             |
| Прогресс-бар уровня    | Плавная анимация заполнения. Цвета: бронза, серебро, золото, платина.                                  |
| Календарь стриков      | 7x5 сетка дней с цветными кружками (зелёный – заход, красный – пропуск, золотой – рекорд).             |
| Ачивки                 | Сетка 3x3 (или горизонтальный скролл). Неполученные – прозрачные/серые.                                |
| Задание дня            | Карточка с текстом и кнопкой «Выполнить» (зелёной, закруглённой).                                      |
| Акции партнёров        | Горизонтальный скролл, каждая карточка – цвет фона из `brand_color_hex`, логотип, название, % кэшбэка. |
| Кросс-селл             | Плитки с иконкой (например, 📈 для Инвестиций) и названием.                                            |
| ИИ-совет               | Блок с иконкой лёгкой лампы, текст, кнопка «Обновить» (иконка обновления).                             |
| Таблица лидеров        | Вкладки, список топ-5 с именами (обезличенными) и суммами, строка с позицией пользователя.             |
| Колесо фортуны         | Круг, разделённый на секторы разных цветов, указатель сверху.                                          |
| Модальное окно приза   | Всплывающее окно с текстом «Вы выиграли X!» и кнопкой закрыть.                                         |

#### 3.6.3. Адаптивность

- Размеры виджетов перестраиваются на мобильных: горизонтальный скролл акций, сетка ачивок становится 2x5, календарь стриков уменьшается.
- На десктопе (≥1024px) может быть двухколоночная сетка: левая колонка (выгода, прогноз, геймификация), правая (акции, кросс-селл, ИИ, колесо).

#### 3.6.4. Тёмная тема

- Все цвета должны быть определены через CSS-переменные, которые переопределяются при переключении темы.
- Пример: `--bg-card`, `--text-primary`, `--accent`, `--border`.

#### 3.6.5. Взаимодействие с разработчиками

- Макеты предоставить в Figma со всеми состояниями (обычное, наведение, выполненное задание, получение ачивки).
- Экспортировать иконки в SVG (предпочтительно) или PNG 2x.
- Анимации описать в формате Lottie или CSS-переходами (плавное появление, вращение).

## 4. Нефункциональные требования

- **Безопасность**: API-ключи (Anthropic API, доступы к БД) передаются строго через переменные окружения `.env` и не хранятся в репозитории.
- **Отказоустойчивость**: при сбоях LLM-провайдера (Claude) или долгой генерации ответа, бэкенд возвращает заглушку, приложение не крашится.
- **Доступность**: Поддержка увеличенных шрифтов.
- **Тёмная тема**: Переключение цветовой схемы без перезагрузки страницы.
