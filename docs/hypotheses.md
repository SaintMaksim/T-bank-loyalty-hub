# Гипотезы для UI (обоснование размещения элементов)

## 1. Виджет «Совокупная выгода» – в самом верху, крупно
Гипотеза: пользователь заходит с вопросом «сколько я выигрываю?».
Реализация: крупная цифра общего эквивалента + разбивка по трём валютам.
Обоснование: пользователи лучше воспринимают одну обобщающую метрику.

## 2. Акции партнёров – горизонтальный скролл
Гипотеза: много акций перегружают экран.
Реализация: 2-3 карточки, остальные листаются вправо.
Обоснование: экономит место, не отвлекает от главного (как в Tinkoff).

## 3. Прогноз – с эмоциональной окраской
Гипотеза: просто цифра не мотивирует.
Реализация: текст «Вы можете получить ~2080 ₽, если продолжите пользоваться картами».
Обоснование: поведенческая экономика – якорение и создание ожидания.

## 4. Геймификация – уровень + прогресс-бар
Гипотеза: пользователи LOW/MEDIUM мотивируются достижением следующего уровня.
Реализация: шкала «Бронза → Серебро → Золото → Платина» и подсказка «Осталось 2900 ₽ до Серебра».
Обоснование: принцип близости цели (goal gradient effect).

## 5. ИИ‑совет – персонализированный по сегменту
Гипотеза: HIGH ценят эксклюзивность, LOW – простые способы сэкономить.
Реализация: текстовый блок «AI совет» с разными сообщениями для каждого сегмента.
Обоснование: повышает релевантность и доверие.

## 6. Кросс-селл – после акций, до геймификации
Гипотеза: после получения выгоды пользователь открыт к другим продуктам.
Реализация: плитки Т‑Инвестиции, Т‑Бизнес, Т‑Мобайл с фильтром по сегменту.
Обоснование: принцип последовательных запросов (foot‑in‑the‑door).
